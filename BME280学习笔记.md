# STM32 + BME280 温湿度气压传感器学习笔记

## 一、项目目标

通过 I2C 驱动 BME280，实现：

- 温度读取
- 湿度读取
- 气压读取
- OLED实时显示
- 驱动模块封装

最终显示：

```c
T:25.51C

H:48.32%

P:1008.01hPa
```

# 二、BME280工作原理

BME280内部包含：

- 温度传感器
- 湿度传感器
- 气压传感器

芯片输出的并不是实际物理量，而是：

```c
ADC原始值
```

需要结合：

```c
校准参数(Calibration Data)
```

进行补偿计算。

# 三、寄存器概念

寄存器本质：

```c
芯片内部的一块存储空间
```

例如：

```c
0xD0
```

芯片ID寄存器。

读取后返回：

```c
0x60
```

表示：

```c
BME280通信正常
```

## 常用寄存器

### 芯片ID

```c
#define BME280_REG_ID 0xD0
```

### 湿度配置

```c
#define BME280_REG_CTRL_HUM 0xF2
```

配置湿度过采样。

例如：

```c
0x01
```

表示：

```c
Humidity x1
```

### 测量控制

```c
#define BME280_REG_CTRL_MEAS 0xF4
```

例如：

```c
0x27
```

表示：

```c
Temperature x1
Pressure x1
Normal Mode
```

### 温度数据

```c
#define BME280_REG_TEMP_MSB 0xFA
```

对应：

```c
0xFA -> MSB
0xFB -> LSB
0xFC -> XLSB
```

组成20位ADC值。

### 湿度数据

```c
#define BME280_REG_HUM_MSB 0xFD
```

对应：

```c
0xFD -> MSB
0xFE -> LSB
```

组成16位ADC值。

### 气压数据

```c
#define BME280_REG_PRESS_MSB 0xF7
```

对应：

```c
0xF7 -> MSB
0xF8 -> LSB
0xF9 -> XLSB
```

组成20位ADC值。

# 四、DIG含义

例如：

```c
dig_T1
dig_P3
dig_H6
```

拆开理解：

```c
DIG
Digital Compensation Parameter
数字补偿参数

T
Temperature

P
Pressure

H
Humidity
```

例如：

```c
dig_T1
=
温度补偿参数1

dig_P5
=
气压补偿参数5

dig_H4
=
湿度补偿参数4
```

# 五、小端模式(Little Endian)

BME280校准参数采用：

```c
LSB在前
MSB在后
```

例如：

```c
0x88 -> LSB
0x89 -> MSB
```

组合：

```c
(msb << 8) | lsb
```

## 踩坑

最开始使用：

```c
BME280_ReadU16()
```

读取湿度参数时出现错误。

原因：

```c
字节顺序理解错误
```

最终改为：

```c
BME280_ReadLE16()
```

专门读取小端数据。

# 六、温度补偿

读取：

```c
adc_T
```

然后按照 Bosch 官方公式计算：

```c
var1
var2
t_fine
```

最终：

```c
T = (t_fine * 5 + 128) >> 8;
```

返回：

```c
0.01℃
```

例如：

```c
2551
=
25.51℃
```

# 七、t_fine的作用

温度补偿过程中得到：

```c
t_fine
```

它不仅用于温度。

还用于：

```c
湿度补偿
气压补偿
```

因此：

```c
static int32_t t_fine;
```

定义为全局静态变量。

流程：

```c
先算温度
得到t_fine

再算湿度
再算气压
```

# 八、湿度补偿

读取：

```c
adc_H
```

读取：

```c
dig_H1
dig_H2
dig_H3
dig_H4
dig_H5
dig_H6
```

按照官方公式补偿。

## H4和H5特殊

手册规定：

```c
H4
H5
跨寄存器存储
```

需要手动拼接：

```c
dig_H4 =
(e4 << 4) |
(e5 & 0x0F);
dig_H5 =
(e6 << 4) |
(e5 >> 4);
```

# 九、符号扩展(Sign Extension)

H4/H5实际为：

```c
12位有符号数
```

需要扩展到：

```c
16位有符号数
```

实现：

```c
static int16_t BME280_SignExtend12(uint16_t value)
{
    if(value & 0x0800)
    {
        value |= 0xF000;
    }

    return (int16_t)value;
}
```

## 原理

12位数：

```c
bit11
```

是符号位。

如果：

```c
bit11 = 1
```

表示负数。

扩展到16位时：

```c
bit15~bit12
```

全部补1。

例如：

```c
1111 1010 0011 0100
```

最终：

```c
(int16_t)0xFA34
```

才能正确表示负数。

# 十、气压补偿

气压是最复杂部分。

使用：

```c
dig_P1~dig_P9
```

和：

```c
adc_P
```

进行计算。

## 为什么要用int64_t

气压公式中存在：

```c
var1 * var1 * dig_P6
```

等运算。

结果可能达到：

```c
几十亿甚至上百亿
```

因此：

```c
int32_t
```

会溢出。

必须：

```c
int64_t
```

# 十一、Q格式定点数

## 湿度

返回：

```c
Q22.10
```

即：

```c
humidity / 1024
```

得到实际湿度。

## 气压

返回：

```c
Q24.8
```

即：

```c
pressure / 256
```

得到实际气压(Pa)。

例如：

```c
25805056
```

转换：

```c
100801 Pa
```

即：

```c
1008.01 hPa
```

# 十二、OLED刷新优化

最开始：

```c
OLED_Clear();
```

放在while循环。

导致：

```c
闪屏
```

原因：

```c
整屏擦除
整屏重绘
```

优化：

不再清屏。

直接覆盖显示。

例如：

```c
OLED_ShowString(...)
```

直接更新。

显示更平滑。

%2ld

# 十三、驱动封装

最终接口：

```c
void BME280_Init(void);

int32_t BME280_ReadTemperature(void);

int32_t BME280_ReadHumidity(void);

int32_t BME280_ReadPressure(void);
```

新增数据结构：

```c
typedef struct
{
    int32_t temperature;
    int32_t humidity;
    int32_t pressure;
}BME280_Data_t;
```

统一读取：

```c
void BME280_ReadData(BME280_Data_t *data);
```

实现：

```c
data->temperature =
    BME280_ReadTemperature();

data->humidity =
    BME280_ReadHumidity();

data->pressure =
    BME280_ReadPressure();
```
